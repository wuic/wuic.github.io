Welcome to WUIC (Web UI Compressor)

The library which minifies your scripts (Javascript/CSS), computes your image's sprites and supports aggregation for you, picking your statics resources everywhere.

The library has been designed to guarantee that good practices around web resources will be always applied when you deploy in production.

Many websites have poor performances due to the many RTTs (round trip time) performed to load images, javascript and css files. A lot of developers know that it is recommanded to aggregate those files to reduce the number of RTTs. Moreover, some tools like YUICompressor can reduce the size of your CSS and Javascript files to speed up the page loading.

However, people do not apply those good practices. The truth is that it is a too expensive task in time in a continuous delivery. People don't have the time to aggregate the files, minify (in case of scripts) or create sprites (in case of images) and change the import statement in there HTML code.

The purpose of WUIC is to help the developer to automate as much as possible these tasks with the maximum of flexibility to guarantee that the device will not be deprecated in the future.

Release

Our current release is the v0.3.2 !

Prior to v0.3.0, WUIC was not on github. See the change logs of previous releases here.

You can integrate the library using this maven dependency (available on the maven central repository) : 

        
        <dependency>
            <groupId>com.github.wuic</groupId>
            <artifactId>wuic-{wuic-module}</artifactId>
            <version>0.3.2</version>
        </dependency>

If your webapp is based on JSPs, the easiest way to use WUIC is to add the wuic-tag artifact in your pom.xml. Otherwise, you can just depend on the wuic-servlet artifact.

Features

WUIC has not implemented all the features in the roadmap yet but it already provides a lot of interesting stuff :

  • CSS & JS compression thanks to YUICompressor
  • Caching thanks to EHCache
  • Embedded image aggregation solution base of the lightmap bin-packing algorithm
  • Automatic sprite generation for aggregated images in both Javascript and CSS languages
  • Access to your static resources in different ways : classpath, webapp, disk, HTTP(S), FTP(S), SSH

Roadmap

WUIC is production ready and is already used in many Capgemini group applications. Just keep in mind that we are young and our solution currently comes with some restrictions :

  • We require the use of EhCache to cache resources
  • We use YUICompressor for CSS & JS compression
  • We aggregate images thanks to the lightmaps bin packing algorithm implemented in the library
We also assume that you won't expect backward compatibility until we release the v1.0.0 of the project. Finally, the project has been essentially tested under Tomcat and Jetty.

The next major release is targeted to the end of July. Intermediate releases are already in the roadmap and more documentation is going to be written in the next weeks ! You can take a look at the milestones for additional information.

We aim to consider WUIC as a real framework when very structural features will be implemented :

  • Chain engines and create custom engines as much as you want
  • Reload automatically configuration and resources at runtime
  • Various cache engine (ehcache, client side, taking application version in consideration, etc)
  • Engine output goes everywhere (media server, HTTP response, etc)
  • Annotation driven configuration with spring
